# Proposal: SNI Proxy Performance Tuning

## 1. 目标 (Objectives)
在保持 L4 转发正确性的前提下，全方位优化代理性能。提升连接处理速度，降低内存分配频率，并最大化吞吐量。

## 2. 核心指标 (Specs)
- **延迟 (Latency)**: 缩短从连接建立到首字节转发的时间。
- **内存 (Memory)**: 减少在高并发下的 GC 压力，实现 Buffer 复用。
- **并发 (Concurrency)**: 提高单机支持的活跃连接数。

## 3. 约束
- 严禁为了性能破坏 TLS 协议的透明性。
- 严禁引入大规模的非必要外部依赖。
- 必须包含性能对比数据（Baseline vs Optimized）。

## 4. 优化领域 (Focus Areas)
- **连接处理**: 精简握手阶段的逻辑。
- **内存分配**: 引入 `sync.Pool` 管理缓冲区。
- **日志渲染**: 避免在高频转发路径上进行繁重的字符串格式化。
