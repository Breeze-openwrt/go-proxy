# Proposal: Implement Asynchronous Zero-delay Pool Refill

## 1. 问题定位 (Problem)
目前的 `BackendPool` 是被动回收制。在高并发突发请求下，预热的连接会被迅速耗尽，后续请求必须等待同步 Dial 建立连接。这是产生“加载瞬间卡顿”的主要原因。

## 2. 改进规格 (Specs)
- **激进回填**: 每当从池中取走一个连接，立即触发一个后台任务去 Dial 一个新连接。
- **并发控制**: 回填任务不应阻塞主请求逻辑。
- **自动收敛**: 如果池子已满（Put 回来的连接填满了），多余的回填连接自动关闭，防止 FD 泄露。

## 3. 交付价值
- 使连接池水位始终保持在“健康线”以上。
- 将突发请求的平均延迟从 RTT 降至物理内存访问级别 (0ms)。
