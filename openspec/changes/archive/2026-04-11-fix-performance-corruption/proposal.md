# Proposal: Fix Performance Issues and Data Corruption

## 1. 问题定位 (Problem)
- **数据损坏**: `BackendPool.Get` 中使用了破坏性的 `Read(1)` 来检查连接存活。这导致每个复用的连接都会丢失首个到达的字节，直接破坏了 TLS 记录或应用协议，导致网页加载极其缓慢或呈现分段现象。
- **转发停顿**: `io.Copy` 在等待 `idle_timeout` 时的阻塞逻辑可能不够平滑。

## 2. 改进规格 (Specs)
- **非破坏性检查**: 使用 `MSG_PEEK` 标志位探测连接状态，绝不消耗任何字节。
- **流完整性保证**: 确保从连接池中取出的所有连接都处于原始流状态。
- **并发吞吐优化**: 优化 `copyWithTimeout` 的循环触发机制，确保数据即到即发。

## 3. 交付价值
- 彻底解决“分段加载”现象。
- 恢复“瞬间打开”的极致体验。
