# Design: Add Daemon Mode

## 1. 后台化实现 (Daemonization)
由于 Go 自身不支持传统的 `fork()` 模型（由于线程安全原因），我们将采用 **Re-exec** 策略：
- 当用户执行 `sni-proxy -d` 时，主进程检测到标志位。
- 启动一个新的子进程，传入非 `-d` 的参数，并设置环境标量或隐藏标志标记为守护态。
- 子进程将工作目录切换至 `/`，并将标准输入输出重定向至 `/dev/null` 或日志文件。
- 父进程随后退出，完成脱离。

## 2. PID 锁与冲突解决 (PID Lock & Conflict Resolution)
1. **检查阶段**: 程序尝试读取指定路径的 PID 文件。
2. **存活判定**: 使用 `syscall.Kill(pid, 0)` 探测进程是否真的存在。
3. **强制接管**: 
    - 如果存活，依次发送 `SIGTERM` 和 `SIGKILL` 确保其退出。
    - 循环等待直到旧进程文件句柄释放。
4. **原子写入**: 写入当前 PID 到文件。
5. **清理钩子**: 在程序 `defer` 或捕获到关闭信号时自动删除 PID 文件。

## 3. 路径适配
优先检查 `/run/` 目录的写入权限，以贴合 `systemd` 的标准行为。
